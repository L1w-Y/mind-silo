## 什么是 App Router

Next.js 13 引入了全新的 App Router，基于 React Server Components 构建，提供了更直观的文件路由系统。

## 文件路由

App Router 使用 `app/` 目录，每个文件夹代表一个路由段：

```
app/
├── page.tsx          # /
├── about/
│   └── page.tsx      # /about
└── blog/
    ├── page.tsx      # /blog
    └── [slug]/
        └── page.tsx  # /blog/:slug
```

## Server Components vs Client Components

默认情况下，`app/` 目录中的组件都是 **Server Components**：

```tsx
// 这是一个 Server Component（默认）
export default async function Page() {
  const data = await fetch("https://api.example.com/data");
  const posts = await data.json();

  return (
    <ul>
      {posts.map((post: any) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
}
```

需要交互性时，添加 `"use client"` 指令：

```tsx
"use client";

import { useState } from "react";

export default function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>Count: {count}</button>;
}
```

## 数据获取

App Router 中推荐直接在 Server Component 中 `await` 数据：

```tsx
async function getData() {
  const res = await fetch("https://api.example.com/posts", {
    next: { revalidate: 3600 }, // ISR: 每小时重新验证
  });
  return res.json();
}
```

## 总结

App Router 让 Next.js 的开发体验更加现代化，Server Components 带来了更好的性能和更简洁的数据获取方式。
