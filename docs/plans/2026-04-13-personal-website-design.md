# 个人网站设计方案

## 一、项目概览

### 定位

以**内容创作和资源管理**为核心的个人网站，通过 OpenClaw 聊天机器人实现碎片化内容的自动收集和分类。GitHub 项目展示作为辅助，不占主要篇幅。

### 核心功能

- 技术博客撰写与展示
- 日常记录（Notion 风格块编辑器）
- 个人资源收藏汇总（URL、文档、技术帖等）
- 碎片灵感/想法记录
- GitHub 仓库与提交记录展示
- OpenClaw 机器人自动收集内容

## 二、技术架构

### 技术选型

| 层面         | 选择                        | 理由                                  |
| ------------ | --------------------------- | ------------------------------------- |
| 框架         | **Next.js (App Router)**    | SSR/SSG/API Routes，Vercel 原生支持   |
| 样式         | **Tailwind CSS**            | 快速实现 Craft 风格的大圆角/留白/阴影 |
| 主题切换     | **next-themes**             | 暗/亮主题无闪烁切换                   |
| Markdown渲染 | **MDX + rehype/remark 插件**| 支持代码高亮、自定义组件              |
| 代码高亮     | **Shiki**                   | VSCode 同款高亮引擎，暗亮主题适配     |
| 全站搜索     | **Flexsearch（客户端）**    | 轻量无后端依赖，构建时生成索引        |
| GitHub 数据  | **GitHub REST API**         | 获取仓库列表和提交记录                |
| 部署         | **Vercel**                  | push 即部署，API Routes 作为 webhook  |

### 数据存储（GitHub 仓库）

```
content/
├── posts/          # 博客文章 (.mdx)
├── journal/        # 日常记录 (.mdx + layout JSON)
data/
├── bookmarks.json  # 资源收藏
├── ideas.json      # 碎片想法
```

## 三、视觉设计风格

### 参考

[Craft.do](https://www.craft.do/zh-CN) — Apple 式极简美学，类似苹果产品 / Notion 的 UI 风格。

### 设计规范

| 要素     | 规范                                                                 |
| -------- | -------------------------------------------------------------------- |
| 配色     | 黑白为主，高对比度，少量强调色点缀；支持暗/亮主题切换                |
| 圆角     | 大圆角（12px-16px），所有卡片、按钮、容器统一                        |
| 阴影     | 柔和弥散阴影，轻浮层次感，不厚重                                     |
| 字体     | 无衬线体，字重层级分明（粗标题 + 细正文）                            |
| 间距     | 宽松留白，内容呼吸感强                                               |
| 动画     | 流畅微交互，hover 跟随、面板滑出、主题切换等过渡自然                 |
| 适配     | 仅桌面端，暂不适配移动端                                             |

## 四、页面设计与交互

### 整体布局

```
                                                ┌──────┐
                                                │ ☀️/🌙 │
                                                └──────┘
      ┌────────┐
      │  首页  │
      │  博客  │
      │  日常  │         内容主区域（长页面，滚动浏览）
      │  收藏  │
      │  想法  │
      └────────┘
```

- **左侧悬浮导航**：竖排胶囊条，垂直居中固定
  - Hover 时高亮背景跟随滑动（滑块跟随效果）
  - 页面滚动时自动同步高亮到当前所在模块
  - Craft 风格：大圆角、柔和阴影、悬浮感
- **右上角**：独立悬浮主题切换按钮，同风格圆润胶囊
- **主区域**：所有模块纵向排列为一个长页面，滚动浏览

### 首页模块

- 简短个人签名 / 一句话介绍
- GitHub 模块：
  - 仓库卡片（星标数、语言标签、描述）
  - 最近提交时间线
- 最新内容聚合：
  - 最近 3 篇博客摘要卡片
  - 最近 5 条收藏卡片
  - 最近几条想法

### 博客模块

- 文章列表：标题 + 日期 + 标签 + 阅读时间估算
- 顶部搜索栏：Flexsearch 即时搜索
- 点击文章 → 右侧滑出阅读面板（带过渡动画）
  - 顶部：文章标题、作者、日期、标签
  - 右侧浮动 TOC 目录导航
  - MDX 渲染正文 + Shiki 代码高亮

### 日常模块

- **块编辑器**布局（Notion 风格）
  - 每条内容为一个 Block
  - 支持拖拽排序
  - 支持调整宽度：全宽 / 半宽 / 1/3 宽
- OpenClaw 上传时自动决定 Block 类型和布局
- 用户可在页面上手动拖拽调整布局

### 收藏模块

- 顶部：分类标签栏（技术、工具、文章、网站等）+ 搜索
- 卡片网格布局：
  - favicon + 标题 + 摘要 + 标签 + 收藏时间
- 点击卡片：新标签页打开原链接

### 想法模块

- 时间线倒序排列
- 短文本直接展示，无需展开
- 每条带标签和时间戳
- 风格轻量

## 五、OpenClaw 机器人对接

### 数据流

```
你发送内容给 OpenClaw
       ↓
OpenClaw AI 分析内容类型
       ↓
  ┌────┴─────────────────────┐
  │ URL/链接 → 资源收藏       │
  │ 短文本/想法 → 碎片灵感    │
  │ 长文本 → 博客草稿/日常    │
  │ 日常内容 → Block 块数据   │
  └──────────────────────────┘
       ↓
调用 Vercel API Route (webhook)
       ↓
API 提交数据到 GitHub 仓库
       ↓
Vercel 检测 push → 自动重新构建部署
```

### API Route 设计

| 接口           | 方法 | 功能                      |
| -------------- | ---- | ------------------------- |
| `/api/collect` | POST | 接收 OpenClaw 推送的内容  |

请求体示例：

```json
{
  "type": "bookmark | idea | post | journal",
  "content": "内容正文或 URL",
  "tags": ["react", "工具"],
  "title": "可选标题",
  "layout": {
    "width": "full | half | third",
    "order": 0
  }
}
```

API 接收后根据 `type` 写入对应文件，通过 GitHub API 提交到仓库。
